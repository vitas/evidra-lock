# P0 Step 3 — Homebrew + Release Alignment

## 1. Decision: packaging strategy

**One archive per binary, one Homebrew formula for `evidra-mcp` only.**

Rationale:
- `evidra-mcp` is the P0 artifact. A developer running `brew install vitas/tap/evidra-mcp` gets exactly one binary that works with zero flags.
- `evidra` CLI ships as a separate archive in the same GitHub Release but does not get a Homebrew formula in P0. Adding one later is a single `brews:` block.
- Separate archives avoid confusion: the `evidra-mcp` tarball contains only `evidra-mcp`, so the formula `install` block is trivial.
- GoReleaser's `brews` section auto-pushes the formula to the tap repo on every tagged release — no manual steps after initial setup.

**Token requirement:** GoReleaser needs a GitHub PAT (`HOMEBREW_TAP_TOKEN`) with `repo` scope to push the formula to `vitas/homebrew-tap`. Store it as a repository secret in `vitas/evidra`.

## 4. GitHub Actions release workflow

**File: `.github/workflows/release.yml`** — full replacement:

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write
  packages: write

jobs:
  # ── Snapshot dry-run (validates GoReleaser config) ──────────────
  snapshot:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: true

      - name: GoReleaser snapshot (dry run)
        uses: goreleaser/goreleaser-action@v6
        with:
          distribution: goreleaser
          version: latest
          args: release --snapshot --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  # ── Docker image ───────────────────────────────────────────────
  docker:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Docker meta
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/vitas/evidra-mcp
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=raw,value=latest

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}

      - name: Smoke test
        run: |
          { cat cmd/evidra-mcp/test/docker/testdata/init.jsonl \
               cmd/evidra-mcp/test/docker/testdata/validate_pass.jsonl; sleep 2; } \
            | timeout 10 docker run --rm -i ghcr.io/vitas/evidra-mcp:latest \
              >stdout.log 2>stderr.log || true

          echo "--- stderr ---"
          cat stderr.log

          grep -q "using built-in ops-v0.1 bundle" stderr.log \
            || { echo "FAIL: missing embedded bundle notice"; exit 1; }

          grep '"id":2' stdout.log | grep -q '"allow"' \
            || { echo "FAIL: response missing allow field"; cat stdout.log; exit 1; }

          echo "PASS: smoke test ok"

  # ── GoReleaser (GitHub Release + Homebrew formula) ─────────────
  goreleaser:
    needs: [snapshot]
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Go
        uses: actions/setup-go@v5
        with:
          go-version-file: go.mod
          cache: true

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          distribution: goreleaser
          version: latest
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}
```

### Changes from current workflow

| What | Before | After |
|---|---|---|
| Snapshot job | missing | validates GoReleaser config before real release |
| GoReleaser depends on | nothing | `needs: [snapshot]` — won't release if config is invalid |
| `HOMEBREW_TAP_TOKEN` | missing | passed to GoReleaser for Homebrew formula push |
| Docker job | unchanged | unchanged |

### Required GitHub secrets

| Secret | Scope | Purpose |
|---|---|---|
| `GITHUB_TOKEN` | automatic | GitHub Release creation, GHCR push |
| `HOMEBREW_TAP_TOKEN` | PAT with `repo` scope | Push formula to `vitas/homebrew-tap` |

To create `HOMEBREW_TAP_TOKEN`:
1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
2. Resource owner: `vitas`
3. Repository access: select `vitas/homebrew-tap`
4. Permissions: Contents → Read and write
5. Copy token → add as secret in `vitas/evidra` repo settings

## 6. Release verification checklist

Run on a clean macOS machine after a tagged release (`v0.1.0`).

### Prerequisites
```bash
# 1. Remove any existing binaries
rm -f $(which evidra-mcp) 2>/dev/null
brew uninstall evidra-mcp 2>/dev/null || true
brew untap vitas/tap 2>/dev/null || true
```

### Install via Homebrew
```bash
# 2. Tap and install
brew tap vitas/tap
brew install vitas/tap/evidra-mcp

# 3. Verify binary is on PATH
which evidra-mcp
# Expected: /opt/homebrew/bin/evidra-mcp (arm64) or /usr/local/bin/evidra-mcp (amd64)

# 4. Verify version output
evidra-mcp --version
# Expected: evidra-mcp 0.1.0 (commit: <hash>, built: <date>)
```

### Zero-config smoke test
```bash
# 5. Run with zero flags — confirm embedded bundle loads
evidra-mcp 2>&1 | head -1
# Expected stderr: "using built-in ops-v0.1 bundle"

# 6. Full MCP round-trip (send init + validate, check response)
{
  echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"test","version":"0.1"}}}'
  echo '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"validate","arguments":{"target":"k8s.pod","action":"delete","namespace":"default","payload":"{}"}}}'
  sleep 2
} | evidra-mcp 2>/dev/null | grep '"id":2'
# Expected: JSON with "allow":true
```

### Checklist summary

- [ ] `brew tap vitas/tap` succeeds
- [ ] `brew install vitas/tap/evidra-mcp` succeeds
- [ ] `evidra-mcp --version` prints semver, commit, date
- [ ] `evidra-mcp` (no flags) prints "using built-in ops-v0.1 bundle" on stderr
- [ ] MCP validate round-trip returns valid JSON with `allow` field
- [ ] Binary is a universal install (works on both amd64 and arm64 macOS)

### Snapshot dry-run (pre-release)

Run locally before tagging to catch config errors:

```bash

ls dist/
```
