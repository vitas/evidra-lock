# AI Development Guidelines

## Purpose
This document defines mandatory governance controls for all AI-assisted work in Evidra-MCP.

## Mandatory Rules
1. All prompts given to AI agents (Codex or any other AI) must be stored in separate files.
2. AI prompts are treated as formal change requests and must be handled with the same control expectations as change input artifacts.
3. Each prompt record must include:
   - Date
   - Author (human actor)
   - Target AI system
   - Intent
   - Full prompt text
   - Expected outcome
4. `ai/AI_{AGENT_NAMW}_PROMPTS_LOG.md` is append-only. Retroactive edits are prohibited.
5. If correction is required, append a new correction entry. Existing entries must not be modified or deleted.
6. All AI-related instruction files must start with the prefix `AI_`.
7. Hidden instructions inside source files are prohibited.
8. Inline prompts inside code are prohibited.
9. Every architectural decision influenced by AI must be documented.
10. AI must not silently refactor code without recording reasoning in `ai/AI_{AGENT_NAMW}_DECISIONS.md`.
11. AI must not restructure directories or rename files without explicit written authorization in a prompt file.
12. AI must not introduce new dependencies without explicit documentation in `ai/AI_{AGENT_NAMW}_DECISIONS.md`.
13. All generated code must include a short comment block with:
   - Generated by AI
   - Date
   - Purpose
   - Related AI prompt file

## Auditability
AI is treated as an execution actor and must be auditable.
All AI-initiated actions must be attributable to a recorded prompt and a corresponding decision trail when applicable.

## Compliance
These controls are mandatory for traceability, auditability, and long-term enterprise maintainability.
